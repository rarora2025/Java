Variables

Declaration vs Initialization vs Assignment

declaration: states the type of a variable, along with its name. A variable can be declared only once. It is used by the compiler to help programmers avoid mistakes such as assigning string values to integer variables. Before reading or assigning a variable, that variable must have been declared

int x

initialization: it’s a special kind of assignment: the first. Before initialization objects have nullvalue and primitive types have default values such as 0 or false. Can be done in conjunction with declaration.

x  = 10 //or...
int x = 10 //both declaration and initalization

assignment: to assign value to a variable. It can be done anywhere, only once with the final-identifier.

x = 20 //already been defined and initalized

Primitive vs Non-Primitive

Primitive Types- created by the programming language

Some examples include: boolean, char and numbers:

  • byte 1 byte -128 to 127.
  • short 2 bytes -32,768 to 32,767.
  • int 4 bytes -2,147,483,648 to 2,147,483,647.
  • long 8 bytes -9,223,372,036,854,775,808 to 9,223,372,036,854,775,80.
  • float 4 bytes 7 decimal digits.
  • double 8 bytes 16 decimal digits.

Non-Primitive Types- made by programmers, also called reference variables

Some examples include:

  • Class
  • Object
  • String (an object)
  • Array
  • Interface

Memory Concepts- destructive vs nondestructive

  • When a value replaces a previous value such as,
    int number1 = input.nextInt();
    the process is said to be destructive

  • When a value is read from a memory location and not destroyed- for example,
    int sum = number1 + number2;
    this process is said to be nondestructive

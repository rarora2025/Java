Recursion

What is Recursion?
The technique of making a function call itself. This technique can be accomplished either directly within the function itself, or indirectly through another method

Recursive method

  • a method calling itself
  • recursive methods would be infinitely long (which would cause a stack overflow error), if they didn’t have some place to escape. Because of this, all recursive methods MUST have a base case.

Halting Condition/Base Case:

  • the condition where the function stops calling itself
  • this is the final run through of the function

In a method is called with a base case it returns a result. Otherwise, it divides the problem into two conceptutal pieces- one which it knows how to do and one which it does not know how to do. They are both identical to each other, but the one that it does know has to do, has a recursive call to the other. The one which it can’t do moves lower in the hierarchy, and when it finally splits into a base case, they return the result up the recursive call hierarchy until it reaches the first piece.

BigDecimal

The BigDecimal class provides operations on double numbers for arithmetic, scale handling, rounding, comparison, format conversion and hashing. It can handle very large and very small floating point numbers with great precision but compensating with the time complexity a bit.
A BigDecimal consists of a random precision integer unscaled value and a 32-bit integer scale. If greater than or equal to zero, the scale is the number of digits to the right of the decimal point. If less than zero, the unscaled value of the number is multiplied by 10^(-scale).

Need Of BigDecimal

  • The two java primitive types(double and float) are floating point numbers, which is stored as a binary representation of a fraction and a exponent.
  • Other primitive types(except boolean) are fixed-point numbers. Unlike fixed point numbers, floating point numbers will most of the times return an answer with a small error (around 10^-19) This is the reason why we end up with 0.009999999999999998 as the result of 0.04-0.03 in the above example.

But BigDecimal provides us with the exact answer.

BigInteger

The BigInteger class is used for mathematical operation which involves very big integer calculations that are outside the limit of all available primitive data types.

For example factorial of 100 contains 158 digits in it so we can’t store it in any primitive data type available. We can store as large Integer as we want in it. There is no theoretical limit on the upper bound of the range because memory is allocated dynamically but practically as memory is limited you can store a number which has Integer.MAX_VALUE number of bits in it which should be sufficient to store mostly all large values.

To perform operations on BigIntegers, use methods BigInteger.add(BigInteger)
BigInteger.divide(BigInteger)

Recursion vs Iteration

The Recursion and Iteration both repeatedly execute the set of instructions. Recursion is when a statement in a function calls itself repeatedly. The iteration is when a loop repeatedly executes until the controlling condition becomes false. The primary difference between recursion and iteration is that recursio* is a process, always applied to a function and iteration is applied to the set of instructions which we want to get repeatedly executed.

Both iteration and recursion are based on a control statement: Iteration uses an iteration statement such as for, while, do...while, and recurses uses a selection statement such as if, if...else, and switch

Both iteration and recursion involve iteration, the difference is that iteration explicitly does it using an iteration statement, whereas recursion achieves it through repeated method calls

Recursion takes up a lot more memory and time, because it has to copy everything over and over again.

Think of Recursion as recursive formulas, think of how you can express element in a set using other members of the set

for example in the fibonnaci sequence, every element f(n) is equal to f(n-1) + f(n-2);

and in the factorials , every element f(n) is equal to n * f(n-1)

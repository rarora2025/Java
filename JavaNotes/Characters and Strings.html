Fundamentals of Characters and Strings

What are character literals?

  • an integer value represented as a character in single quotes. For example, 'z' represents the integer value of the letter z, and '\t' represents the integer value of a tab character.

What are String literals?

  • a sequence of characters in double quotation marks; stored in memory as a String object
  • may be assigned to a String reference; For example
    String color = blue;

String Methods

There are three major String methods:

  • length
  • charAt
  • getChars

To use length, attach the .length() suffix to the end of a String, which will return the length of a String

To use charAt, attach the .charAt(int) suffix with a parameter stating the index of which character in the String you’d like.

The method getChars() has four parameters, and it is ued to copy characters from a string to a char array. Like the other methods, it can be utilized by attaching the suffix to the end of a String, with the four parameters as follows:
Parameter 1: starting index of characters being copied
Parameter 2: one past the index of the last character you want to be copied
Parameter 3: the character array that you want the characters to be copied into
Parameter 4: the starting index where the copied characters are placed into in the new array

getChars(startIndex, endIndex, targetArray, targetArrayinsertIndex)

Comparing Strings

Methods:

  • .equals()
  • .equalsIgnoreCase()
  • .compareTo()
  • .regionMatches()

.equals() compares the values (using a lexicographical comparison) stored into variables, whereas == compares the references of the variables- or the address in memory. .equals() returns true if the values are equivalent, and false if they are different

Syntax:
s1.equals(s2);

The equalsIgnoreCase() method compares two strings irrespective of the case (lower or upper) of the string. This method returns true if the argument is not null and it represents an equivalent String ignoring case, else false.

Syntax:
str2.equalsIgnoreCase(str1);

compareTo() is used for comparing two strings lexicographically. Each character of both strings are converted into a Unicode value. However, if both the strings are equal, then this method returns 0 else it only result either negative or positive value.

Syntax:
str2.compareTo(str1);

The startsWith() method is used for checking prefix of a String. It returns a boolean value true or false based on whether the given string begins with the specified letter or word.

For example:
String str = "Hello";
//This will return true because string str starts with "He"
str.startsWith("He");

– If you want to check if a word starts with something, but not directly at the beginning, use the index parameter, after the prefix you are checking. For example if you want to see if the word “Hello” starts with “el” at the second position, do this:
str.startsWith("el", 1);

Java String endsWith(String suffix) method checks whether the String ends with a specified suffix. This method returns a boolean value true or false. If the specified suffix is found at the end of the string then it returns true else it returns false.

For example:
String str = "Hello";
//This will return true because string str ends with "lo"
str.endsWith("lo");

IndexOf() and lastIndexOf()

IndexOf() returns the index of the first occurrence of a character/substring in a given String or part of a string.

The syntax of the method is as follows:

x = character/substring you are looking for in “String”
y = position in “String” where you should start searching for x

String.indexOf(x, y)

The lastIndexOf() method works the same way. It returns the position of the last occurrence of specified character/substring in a string/part of a string.

For example,

String myStr = "Hello planet earth, you are a great planet.";
System.out.println(myStr.lastIndexOf("planet"));

Substring Methods
The substring methods extract certain characters from a string:

//this will extract characters 1-3 (doesn't do last index)
var  str =  "Hello world!";  
var  res = str.substring(1,  4);

Concatenating Methods
The concat() method concatenates one string to the end of another string. This method returns a string with the value of the string passed into the method, appended to the end of the string.

For example:

String s = "Gfg";
s = s.concat("! is the best.");
System.out.println(s);

or

String h = "Happy";
full = h.concat(" Birthday");
System.out.println(full);

  • toUpperCase(): makes the entire String uppercase

  • toCharArray(): converts a String into an array (char) containing the letters of that word

valueOf() Method

  • basically the same thing as the toString() method

-String.valueOf(Object) will return null, whereas Object::toString() will throw a null pointer exception.

  • valueOf() method of String class is static (doesn’t need an object to be called on), whereas toString() method of String class is non static.

Buffers
Imagine that you’re eating candy out of a bowl. You take one piece regularly. To prevent the bowl from running out, someone might refill the bowl before it gets empty, so that when you want to take another piece, there’s candy in the bowl.

The bowl acts as a buffer between you and the candy bag.

If you’re watching a movie online, the web service will continually download the next 5 minutes or so into a buffer, that way your computer doesn’t have to download the movie as you’re watching it (which would cause hanging).

How to Create a new StringBuilder Buffer

  • create a new StringBuilder Object and name it buffer.
    –There are 4 constructors/different ways you can create this; three of them are as follows:

  1. StringBuilder() // will create a Builder with an initial capacity of 16 characters

  2. StringBulder(10) // will create a Builder w/ an initial capacity of the 10 characters

  3. StringBuilder("Hello") // will create a Builder with an initial capacity of 16 + number of characters in the String given- 21

StringBuilder Methods

  • buffer.length(): returns number of chars currently in String builder
  • buffer.capacity(): returns number of characters that can be stored without allocating more memory
  • buffer.ensureCapacity(int): expands the capacity of a String builder to at MINIMUM the specified number
  • buffer.setLength(int target): either removes trailing characters until number of characters is the target number, or adds null characters until number of characters is the target number

Manipulating Characters

  • buffer.charAt(int index): returns the character of the StringBuilder at the given index

  • buffer.setCharAt(int index, char new char): takes the char at specified index of String builder, and replaces it with the new char (argument) given.

  • buffer.getChars(int srcBegin, int srcEnd, char[] dst, int dstBegin): copies the characters starting at the given index:srcBegin to index:srcEnd-1 from String contained by StringBuilder into an array of char passed as parameter to function.

  • buffer.reverse(): reverses the contents of the StringBuilder.

  • buffer.insert(int index_position, thingtoinsert): inserts the string representation of given data type at given position

  • buffer.delete(int startindex, int endindex): removes the characters in a substring of this StringBuffer. The substring begins at the specified start and extends to the character at index end - 1 or to the end of the StringBuffer if no such character exists.

  • buffer.deleteCharAt(int index): removes the char at the specified position in this sequence

  • buffer.append(): add something to the end of a String builder

– if you want to add a new line in a string, append a System.getProperty("line.seperator") to the end of a String builder
